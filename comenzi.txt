// exec: executa ceva intr-un container deja ruland (mongos), -i stdin deschis, (-t): pseudo-terminal
- -i si -t sunt optiuni cum se leaga terminalul meu de procesul pornit in container
 -i: (sau interactive): redirect. inputul catre procesul din container (fara -i, programul 
s-ar fi inchis imediat / nu primea nimic de pe stdin)
 -t: face procesul din container sa creada ca ruleaza intr-un terminal real 
 // stdout si stderr ale procesului din container sunt redirectionate catre terminalul tau

// mongosh: comanda executata in containerul tinta (mongos), --port, portul la care se conect. mongosh
( e pornit la portul 26061 )
// --eval: executa expresia / codul dat ca argument (aka: porneste mongosh, executa comanda, afis. rez)
